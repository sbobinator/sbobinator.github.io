Tecnologie Core

NeMo Parakeet

Il modello TDT 0.6B v3 di NVIDIA offre riconoscimento vocale italiano-first con timestamp sui segmenti. Caricato una volta per processo worker, gira su CPU per accessibilità o CUDA quando serve velocità su archivi lunghi.

FastAPI + HTMX

UI da operatore leggera senza SPA pesante. Form upload, liste job e pagine progress per job interrogano il server per aggiornamenti live. Template Jinja2 mantengono lo stack semplice da deployare e verificare.

Docker Compose

Impacchetta UI e worker per mini PC Linux headless. Il modello ASR può essere incluso nell'immagine; Qwen locale si scarica all'avvio se la RAM lo consente. Mapping porta default 8502:8501 per home-lab.

Il Worker in Background.

NeMo non può caricarsi in sicurezza nel thread delle richieste del web server su Windows — quindi la trascrizione gira in un processo worker dedicato. Prende il prossimo job in coda, esegue la pipeline completa e ripete fino allo stop.

All'avvio il worker recupera job running orfane e riconcilia cartelle su disco con SQLite — così un crash a metà trascrizione non lascia la cronologia in uno stato incoerente.

  • Recupero orfani al boot
  • Riconciliazione disco ↔ DB
  • ASR a chunk per file lunghi
worker.py 
while not _stop_event.is_set():
    job = claim_next_job()
    if job is None:
        _stop_event.wait(poll_interval)
        continue

    logger.info("Elaborazione job %s", job.id)
  try:
        run_pipeline(job.id)
  except Exception:
        logger.exception("Job %s fallito", job.id)
